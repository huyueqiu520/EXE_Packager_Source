# EXE打包工具使用说明

## 软件概述

EXE打包工具是一款图形化界面的Python转EXE工具，基于PyInstaller开发。它能够将Python脚本打包成独立的Windows可执行文件，无需用户掌握复杂的命令行操作。

## 系统要求

- Windows 7及以上版本
- Python 3.6或更高版本
- PyInstaller库

## 安装依赖

在使用本工具前，请确保已安装PyInstaller：

```bash
pip install pyinstaller
```

## 软件功能

### 主要功能

1. **图形化界面**：直观的操作界面，简化打包流程
2. **单文件打包**：将Python脚本及其所有依赖打包成单个exe文件
3. **窗口模式**：可选择打包为窗口程序，运行时不显示控制台
4. **自定义图标**：支持为生成的exe文件设置自定义图标
5. **指定输出目录**：可自定义打包结果的存放位置
6. **实时日志**：显示打包过程的详细日志信息

### 界面说明

界面分为三个主要区域：

1. **文件选择区域**
   - Python文件：选择要打包的Python脚本
   - 输出目录：设置打包结果的存放位置

2. **打包选项区域**
   - 单文件模式：勾选后将所有内容打包到一个exe文件中
   - 窗口模式：勾选后运行时不显示控制台窗口
   - 图标文件：选择ico格式的图标文件

3. **打包日志区域**
   - 实时显示打包过程中的日志信息

## 使用步骤

### 基本使用流程

1. **启动程序**
   - 运行 `python exe_packager.py` 启动工具

2. **选择Python脚本**
   - 点击"浏览"按钮，选择要打包的Python文件
   - 程序会自动填充输出目录为脚本所在目录

3. **设置输出目录**
   - 可使用默认目录或点击"浏览"选择自定义目录

4. **配置打包选项**
   - 根据需要勾选"单文件模式"和"窗口模式"
   - 可选择自定义图标文件

5. **开始打包**
   - 点击"开始打包"按钮
   - 等待打包完成

6. **查看结果**
   - 打包完成后，生成的exe文件会存放在指定输出目录
   - 状态栏会显示打包结果

### 打包选项详解

- **单文件模式**
  - 优点：生成单一exe文件，便于分发
  - 缺点：启动速度稍慢，占用更多内存
  - 适合：需要简单分发的场景

- **窗口模式**
  - 用于打包GUI应用程序
  - 打包后的程序运行时不显示控制台窗口
  - 适合：图形界面应用程序

- **自定义图标**
  - 需要ico格式的图标文件
  - 生成的exe文件将使用指定图标

## 常见问题

### 1. 打包失败怎么办？

检查以下几点：
- Python脚本本身是否有语法错误
- 是否安装了PyInstaller
- 依赖库是否正确安装
- 路径中是否包含中文或特殊字符

### 2. 打包后的exe文件过大？

- 尝试不使用单文件模式，生成文件夹形式
- 检查是否包含了不必要的依赖
- 部分库可能会影响最终文件大小

### 3. 运行exe文件报错？

- 确保目标机器安装了Visual C++ Redistributable
- 某些库可能需要额外的系统组件
- 尝试在控制台模式下运行，查看具体错误信息
